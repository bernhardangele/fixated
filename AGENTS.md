# AGENTS.md

## Project Overview

`fixated` is an R package for processing and analyzing eye movement data recorded by eye trackers such as the SR Research EyeLink system. This repository is in early development.

## Repository Structure

Standard R package layout:

```
fixated/
├── .github/                  # GitHub configuration and workflows
├── R/                        # R source files
├── tests/                    # Test files (testthat framework)
│   └── testthat/
├── man/                      # Auto-generated documentation (roxygen2)
├── vignettes/                # Long-form documentation
├── DESCRIPTION               # Package metadata
├── NAMESPACE                 # Exported functions (auto-generated by roxygen2)
└── README.md                 # Package overview
```

## Features

- **Read raw data** – Parse EyeLink ASC files (`read_asc()`) or generic CSV files (`read_samples_csv()`) into tidy data frames.
- **Detect fixations** – Identify fixations from raw gaze samples using the dispersion-threshold identification (I-DT) algorithm (`detect_fixations()`).
- **Clean fixations** – Merge short adjacent fixations and remove outliers based on duration or spatial bounds (`clean_fixations()`).
- **Read regions of interest** – Load word-level bounding boxes from a CSV file (`read_roi()`).
- **Compute eye-movement measures** – Calculate standard reading-research measures per word (`compute_eye_measures()`): FFD, GD, GPT, TVT.

## Coding Conventions

- Follow the [tidyverse style guide](https://style.tidyverse.org/).
- Document all exported functions using `roxygen2` comments (`#'`).
- Use `snake_case` for function and variable names.
- Keep functions small and focused; prefer many small functions over large monolithic ones.
- Use `testthat` for all new functionality.
- Use the `usethis` package for boilerplate (e.g., `usethis::use_r()`, `usethis::use_test()`).

## Building and Testing

```r
# Install dependencies
devtools::install_deps()

devtools::document()

devtools::test()

devtools::check()
```

## Workflow: Adding New Features

1. Create or edit the appropriate `.R` file in the `R/` directory. Use `usethis::use_r("feature_name")` for new components.
2. Add `roxygen2` documentation above the function.
3. Run `devtools::document()` to update `NAMESPACE` and `man/`.
4. Create a test file with `usethis::use_test("feature_name")`.
5. Run `devtools::test()` to verify all tests pass.
6. Run `devtools::check()` to ensure no R CMD check warnings or errors.

## Coordinate System

All spatial coordinates follow the **EyeLink convention**: (0, 0) is the **top-left corner** of the display, with x increasing rightward and y increasing downward.

## Pull Request Guidelines

- Keep changes small and focused.
- Include tests for all new functionality.
- Ensure `devtools::check()` passes with no errors or warnings before submitting.
- Update `NEWS.md` (when it exists) with a brief description of changes.
